api.phantom.app/image-proxy (lab)

Phantom’s backend fetches each image= target (server-side), then your harness loads the proxied bitmap with <img>. So http://localhost:… hits their loopback / rejects — not your dev machine unless you expose a public URL (tunnel). Public HTTPS icons (PNG or SVG rasterized via the proxy) behave like CookieYes-style CDNs. fetch() to api.phantom.app may return 200 when CORS permits your origin — check Network / tape.
